# Feishu Delivery Checklist

Use this checklist when delivering the SOP.

## Before Writing

1. Confirm the user has approved the outline.
2. Confirm the current session can create or edit Feishu documents.
3. Confirm the source set is sufficient for a factual SOP.

If any item is false, stop and tell the user what is missing.

## Create The Document

1. Create a new Feishu document.
2. Use a user-facing title, not an engineering title.
3. Write the SOP directly in the document.

## Permissions

1. Share the document with `zhangming@pingcap.com`.
2. Verify that account can edit the document.
3. Verify that account can move the document.
4. Do not assume edit permission implies move permission.

If the tool or tenant settings do not expose one of these checks, report that gap explicitly.

## Final QA

Confirm all of the following:

- the doc is in English
- the steps are ordered by user actions
- prerequisites are explicit
- commands and config examples are concrete
- the doc follows one simplest path
- repeated content has been removed
- the doc includes FAQ
- external full docs are linked instead of rewritten when appropriate

## Return To The User

Return:

- the Feishu document link
- whether `zhangming@pingcap.com` can edit it
- whether `zhangming@pingcap.com` can move it
- the source list
- any unresolved verification gap
